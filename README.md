<h1 align="center">Ёфикатор <a href="https://github.com/zapeko/YoRZ">YoRZ</a></h1>

<p align="center">
  <b>Умный и настраиваемый инструмент для автоматической и полуавтоматической расстановки буквы «ё» в русском тексте.</b>
</p>

## О проекте

Логика работы программы кардинально отличается от большинства существующих ёфикаторов: **YoRZ расставляет букву «ё» вместо «е» во всех словах, если этих слов нет в его специализированных словарях исключений**. 

Программа построена на строгой последовательной обработке текста с использованием четырёх уровней словарей. Итоговый результат сохраняется не только в чистом виде, но и в виде HTML-файла с цветовой подсветкой всех произведённых замен — это позволяет быстро визуально проверить корректность работы словарей.

Поддерживаемые форматы файлов: **`.txt`**, **`.md`**, **`.fb2`**, **`.epub`**.

---

## 📚 Логика работы словарей

Словари применяются строго в следующем порядке:

1. 🔵 **Синий словарь (`blue.dic`)** — *Автоматически*. Словарь исключений (регулярные выражения без буквы «ё»).
2. 🟢 **Зелёный словарь (`green.dic`)** — *Автоматически*. Словарь автозамены (регулярные выражения с буквой «ё»).
3. 🟠 **Оранжевый словарь (`orange.dic`)** — *Интерактивно*. Словарь слов-омографов (слова разного значения, например «все» и «всё»). Программа останавливается и предлагает пользователю выбрать правильный вариант, исходя из контекста.
4. 🟡 **Жёлтый словарь (`yellow.dic`)** — *Автоматически*. Корневой (основной) словарь базы слов.

**Особенности обработки:**
- Участки текста, обработанные синим словарём, становятся «невидимыми» (исключаются) для проверки зелёным словарём.
- Участки, обработанные синим или зелёным словарем, исключаются из проверки оранжевым.
- Обработка жёлтым словарём сканирует **весь текст целиком**, включая фрагменты, ранее изменённые другими словарями.

## ✨ Основные возможности

- **Современный графический интерфейс (GUI):** Встроенные светлая и тёмная темы, настройка шрифтов и прозрачности подсветки, перенаправление вывода терминала прямо в окно программы.
- **Поддержка разметки:** Безопасная работа с форматами FB2, EPUB и Markdown. Программа автоматически распознаёт и защищает от искажения ссылки, email-адреса, названия файлов и HTML-коды цветов.
- **Интерактивность и безопасность:** Мгновенная остановка длительного процесса ёфикации (кнопкой в GUI или `Ctrl+C` в консоли) с **сохранением прогресса** для последующего продолжения.
- **Автообновления:** Проверка новых версий и обновление программы прямо из интерфейса (в версии для Windows).
- **Синхронизация словарей:** Возможность скачивания актуальных словарей с GitHub в один клик и бережного слияния с пользовательскими словарями.

## 📥 Установка и запуск

### Для пользователей Windows (Рекомендуемый способ)
Скомпилированная портативная версия со встроенным GUI, работающая «из коробки» (не требует установки Python).

1. Перейдите в раздел **[Releases](https://github.com/zapeko/YoRZ/releases)** и скачайте последнюю версию установщика `YoRZ.exe`.
2. Запустите файл и следуйте инструкциям на экране.
3. Откройте программу с ярлыка на рабочем столе.

> **Важно:** Программа хранит ваши личные настройки и изменения в словарях в безопасной системной папке (`%APPDATA%\YoRZ`). При установке новых версий **ваши правки в словарях никогда не затрутся**!

### Для Linux / macOS / Разработчиков (Из исходного кода)

Для работы потребуется **Python 3.8+**.

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/zapeko/YoRZ.git
   cd YoRZ
   ```
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Запустите нужный интерфейс:
   - **Графический интерфейс (GUI):** `python gui.py`
   - **Консольное меню (CLI):** `python main.py`

*(При первом запуске из исходников программа автоматически создаст папку для словарей пользователя в системной директории: `~/.config/YoRZ` на Linux или `~/Library/Application Support/YoRZ` на macOS).*

## 🛠 Руководство: Пошаговая работа с текстом

Программа разбита на отдельные модули для тонкой настройки базы и подготовки текста. Для достижения идеального результата рекомендуется строго соблюдать последовательность:

### Шаг 1. Типограф (Подготовка текста)
Тексты из интернета часто содержат мусор: лишние пробелы, мягкие переносы, дефисы вместо тире. Типограф приводит текст к стандарту, что критически важно для работы регулярных выражений. Дополнительно он игнорирует десятичные дроби и веб-ссылки, защищая их от искажения.
- **Результат:** Создаётся чистый файл `_fixed` (например, `book_fixed.txt`), который используется на следующих этапах.

### Шаг 2. Извлечение слов
Программа сканирует текст и выписывает все слова с буквами «е» или «ё», которых ещё нет в вашей базе слов (`yellow_base.txt`).
- **Результат:** Файл `_extraction.txt`. Проверьте этот файл, расставьте «ё», где требуется. Слова с пометкой `(!)` означают, что в базе уже есть похожее слово. Если оно верное, уберите пометку, если нет — удалите строку. После ручной проверки добавьте эти новые слова в `yellow_base.txt`.
- *Бонус:* Слова с пометкой `(!)` также дублируются в `for_blacklist.txt`. Их можно добавить в `blacklist.txt` для игнорирования в будущем.

### Шаг 3. Сортировка базы
Автоматически сортирует файл `yellow_base.txt` по алфавиту и удаляет дубликаты. Это очищает базу от мусора, который мог появиться на предыдущем шаге.
- **Результат:** Отсортированный файл базы.

### Шаг 4. Поиск омографов (Двойников)
Программа ищет в базе слова, которые пишутся одинаково без «ё», но имеют разный смысл (например, *все/всё*, *мел/мёл*).
- **Результат:** Новые пары будут автоматически добавлены в словарь `orange.dic` под комментарием "Добавлено автоматически".

### Шаг 5. Формирование финального словаря
Ёфикатор не читает текстовую базу напрямую — ему нужен оптимизированный словарь регулярных выражений. Инструмент автоматически собирает финальный `yellow.dic`, объединяя неизменяемые корни (`yellow_root.txt`), вашу пополняемую базу (`yellow_base.txt`) и дополнения (`yellow_add.txt`).
- **Результат:** Готовый к работе `yellow.dic`.

### Шаг 6. Ёфикация текста
Финальный этап расстановки буквы «ё» с применением всех четырёх словарей к вашему тексту. Во время работы программа будет останавливаться на словах-омографах из оранжевого словаря, ожидая вашего выбора контекста.
- **Результат:**
  1. Размеченная HTML-версия `_yo.html` с удобной цветовой подсветкой (синий, зелёный, оранжевый, жёлтый) для визуальной проверки замен.
  2. Чистая версия с суффиксом `_yo` в формате, соответствующем формату входного файла (.txt, .md, .fb2 или .epub).

> Если текст очень большой, вы в любой момент можете **приостановить процесс**. Программа безопасно сохранит прогресс, и при следующем запуске вы сможете продолжить работу с места остановки!
